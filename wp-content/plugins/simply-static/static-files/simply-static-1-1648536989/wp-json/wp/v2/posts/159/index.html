Sebagai developer yang kesehariannya nggak pernah lepas dari bagaimana caranya mengembangkan sebuah software dan aplikasi, kamu pasti udah familiar banget sama salah satu sistem database berbasis NoSQL. Salah satu database berbasis NoSQL yang paling digemari oleh para developer adalah Firebase. Selama beberapa tahun terakhir perkembangan Firebase udah semakin terkenal di kalangan developer dunia, termasuk Indonesia loh kisanak!
<\/p>\n\n\n\n

Walaupun banyak di antara developer yang udah pakai Firebase, pasti masih banyak juga nih yang belum tau apa itu Firebase. Kebetulan banget hari ini Qatros Teknologi Nusantara mau bahas tentang Firebase besutan dari Google. Yuk simak yuk!
<\/p>\n\n\n\n

Apa itu Firebase?<\/strong>
<\/p>\n\n\n\n

Seperti yang udah Qatros sebutkan di atas, Firebase adalah sebuah layanan baru yang dirilis oleh Google. Firebase merupakan sebuah sistem database berbasis NoSQL yang dibuat Google dengan tujuan untuk memberikan kemudahan untuk developer aplikasi dalam proses mengembangkan aplikasi mereka.
<\/p>\n\n\n\n

Firebase yang merupakaan BaaS atau Backend as a Service ini bisa dibilang adalah sebuah solusi dari Google untuk mempercepat pekerjaan developer aplikasi loh! Kenapa kok gitu sichhh? Ya karena dengan hadirnya Firebase ini, para developer nggak perlu jadi super effortfu<\/em>l untuk ngurusin urusan backend.
<\/p>\n\n\n\n

Penasaran nggak sih kapan Firebase ini ditemukan dan dibuat sama siapa ya kira-kira?<\/strong>
<\/p>\n\n\n\n

Nggookhey kisanak kita mulai perjalanan sejarahnya, Firebase ini pertama kali didirikan oleh Andrew Lee dan James Tamplin pada tahun 2011 dengan Realtime Database sebagai produk pertamanya. Sebagai produk pertamanya, Realtime Database digunakan developer untuk menyimpan data dan melakukan sinkronisasi ke banyak user sekaligus. Lambat laun, Realtime Database berkembang menjadi sebuah layanan pengembangan aplikasi. Sampai pada Oktober 2014, perusahaan besutan Andrew dan James ini diakuisisi oleh Google.
<\/p>\n\n\n\n

Hingga saat ini, Firebase udah punya dua pilihan layanan yang bisa kamu pakai. Dua layanan ini adalah SPARK (untuk menggunakan layanan secara gratis) dan BLAZE (untuk menggunakan layanan premium berbayar). Siapa di sini yang udah coba Firebase?
<\/p>\n\n\n\n

Apa Aja Sih Jenis dan Fitur yang dimiliki Firebase?<\/strong>
<\/p>\n\n\n\n

1. Firebase Analytics<\/strong>
<\/p>\n\n\n\n

Mirip-mirip sama Google Analytics, Firebase Analytics ini adalah salah satu fitur Firebase yang digunakan sebagai koleksi data dan reporting untuk aplikasi Android dan iOS. Kisanak bisa banget mengolah data yang ada di sini untuk dibuat laporan. Selain itu dengan fitur ini kisanak bisa membuat segmentasi user berdasarkan user attribute loh!
<\/p>\n\n\n\n

2. Firebase Cloud Messaging dan Notifications<\/strong><\/p>\n\n\n\n


<\/strong>Firebase Cloud Messaging atau yang biasa dikenal sebagai FCM adalah fitur Firebase yang menyediakan koneksi antar server dan device. Dengan Firebase Cloud Messaging, kisanak bisa mengirim dan menerima pesan. Nggak cuma itu aja, dengan Firebase Cloud Messaging juga memungkinkan untuk memberikan notifikasi di device dan web tanpa harus memakan biaya.<\/p>\n\n\n\n

3. Firebase Authentication<\/strong>
<\/p>\n\n\n\n

Kalau yang satu ini adalah salah satu layanan backend untuk fitur Android dan iOS. Fitur ini berfungsi untuk melakukan autentifikasi pengguna ke aplikasi yang dibuat oleh developer. Firebase Authentication ini mendukung autentifikasi menggunakan nomer telepon, password, dan lain-lain. Selain itu Firebase Authentication juga terintegrasi dengan fitur layanan Firebase lainnya. Paket lengkap nih! 
<\/p>\n\n\n\n

5. Firebase Cloud Firestore<\/strong>
<\/p>\n\n\n\n

Fitur lainnya yang dimiliki Firebase adalah Firebase Cloud Firestore yang merupakan sebuah database fleksibel dan terukur untuk pengembangan perangkat. Dengan fitur Firebase Cloud Firestore, kamu bisa membuat data tetap terkoneksi di aplikasi user menggunakan listener realtime.
